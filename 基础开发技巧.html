2.如何新建一个数据管理页面

2.1.首先新建一个 .cfg 文件，格式模板如下

XXXX 配置文件

表名称
TABLENAME: XXX

主键名称
TABLEKEY:col_2,col_3,col_5

应选列名(所有列)
SELECTEDCOLUMN:col_1,col_2,col_3,col_4,col_5,col_6

数据类型(所有列)
DATATYPE:VARCHAR,VARCHAR,NUMBER,VARCHAR,NUMBER,VARCHAR

主键索引(从1开始)
KEYINDEX:2,3,5

显示列名(所有列)
HEADERNAME:列名_1,列名_2,列名_3,列名_4,列名_5,列名_6

隐藏列号(第一个必须是0)
HIDENCOLUMN:0,6

自动填充字符(填充空格就直接打space)
AUTOFILLTEXT: , ,0,a, ,

3.代码文件结构

QT 的设计遵循一个原则：一个界面就是一个类

3.1.基本结构

界面基本结构如下所示

Widget.h
    |
    +-- Widget.cpp 
    |      同名的 cpp 中一般存放构造函数，析构函数
    |
    +-- Widget_Construct.cpp
    |      Construct.cpp 中存放 void Construct() 函数
    |      主要用来设置整个界面的初始化布局
    |
    +-- Widegt_Add/Delete/Change/Find.cpp
    |      主要存放对于数据库表的增/删/改/查功能的实现
    |
    +-- Widget_Save.cpp
    |      存放 void Save() 函数，用来对用户的增删改查
    |      进行保存，使得结果持久化在数据库中
    |
    +-- Widget_MessageBox.cpp
    |      存放弹出的对话框信息，一般为三种：告知型/警告型/错误型
    |
    +-- Widget_ShowData.cpp
           存放 void ShowData() 函数，主要用来从数据库中读取数据
           并将数据显示在表格中

3.2.界面控件定义

在 Widget.h 中定义所有需要的控件

必须要以指针形式，从堆上 new 出对象，

从栈上实例化的控件对象将无法显示在界面上

3.3.ConstInfo 与 Tool

ConstInfo 和 Tool 是两个比较特殊的类这两个类都是以单例模式定义的，类可以被多次实例化，但所有类对象均指向同一个地址

ConstInfo 一般用来存放全局常量，根据实际需求进行改造

Tool 一般永海存放一些工具函数，比如读取文件，分割字符串等等

一般来说，所有的界面的数据成员都会有这两个类，在构造函数中实例化即可

实例化方法：

// 在头文件中
class Widget
{
private:
    Tool *tool;
    ConstInfo *const_info;
};

//在源文件中
Widget()
{
    tool = Tool::getInstance();
    const_info = ConstInfo::getInstance();
}
